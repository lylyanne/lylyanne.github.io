Ruby class concept

7 September 2014

Ruby is object oriented programming language.

In Ruby, you can use a class to organize your object's attributes and methods.

Below is an example of simple Ruby class:

class BankAccount 
	def initialize(name) 
		@name = name 
		@balance = 0 
	end 

	def deposit(amount) 
		@balance += amount 
	end 

	def withdraw(amount)
		sufficient_fund_check(amount) ? @balance -= amount : puts "Insufficient amount"
	end

	private
	def sufficient_fund_check(amount)
		@balance >= amount
	end
end

Per convention, class name should start with a capital letter and use CamelCase instead of snake_case. Method name, on the other hand, should be in lowercase and use snake_case.

You can create a new instance of a class like below:

jack = BankAccount.new("jack")
jill = BankAccount.new("jill")
<

You can access the class methods as below:

jack.deposit(200)
jill.deposit(500)

jack.withdraw(50)
jill.withdraw(150)

In the class example above, @name and @balance are instance variables. That means, @name and @balance value will be different from one object to another. @balance of jack will be different than @balance value of jill.

Besides instance variable, Ruby class can have 3 other types of variables:

  • Global variable: This variable is preceded by $ sign. This variable is available across class.
  • Local variable: This variable is available only within a particular method.
  • Class variable: This variable is preceeded by double @sign. This variable is available across different objects. A good example is @@accounts, which is a variable to see how many total Bank Accounts are open.

Regarding methods, Ruby offers public, private, and protected methods. By default, Ruby methods are public.

When you declare the word private (or protected) somewhere inside a Ruby class, all methods after that declaration will be of that type unless a declaration of another type is encountered by Ruby.

The difference between the three methods:

  • A public method is a method that is accessible to anyone.
  • A private method is a method that cannot be called with an explicit receiver and only accessible from within the defining class. A method is usually declared as private method when there is no specific reason for the user to access it directly, as in the case of sufficient_fund_check above.
  • A protected method is a method that can be called with an explicit receiver as long as the receiver is self or object of the same class as self. You can read more about it here.
