Blocks, Procs, and Lambdas

14 September 2014

Blocks

A block is just pieces of code between the opening and closing curly bracket or between the do and end keyword.

The purpose of block is to instruct Ruby on how to process each element in an Array or Hash.

For example, below code are using the same method but the result will be different since different blocks are provided.

arr = [1,2,3,4,5]
arr.each do |x| 
	p x * 2
end #=> 2,4,6,8,10

arr.each do |x|
	p x if x % 2 == 0
end #=> 2,4

Procs

A proc is a block of code that can be saved and assign to a variable. An example of proc is:

three_letter_word = Proc.new do |x|
x if x.length == 3
end

["cat", "tiger", "lemur", "dog", "monkey"].select(&three_letter_word) #=> ["cat", "dog"]

You see in the select method how the &three_letter_word is replacing the do..end block from three_letter_word above? The ampersand(&) symbol is the convention we use to convert our proc back to block.

Proc can also be called by using the call method like below:

letter_builder = Proc.new { |x| puts x }

letter_builder.call(["Dear Friend,", " Just want to say I am thinking of you", " Hope you have a wonderful day today", "Your BFF"])
#=>Dear Friend,
# Just want to say I am thinking of you
# Hope you have a wonderful day today
# Your BFF

Lambdas

A lambda is created for the same purpose as proc though its syntax is different as you can see below.

three_letter_word = lambda { |x| x if x.length == 3 }
["cat", "tiger", "lemur", "dog", "monkey"].select(&three_letter_word) #=> ["cat", "dog"]

A lambda, just like proc, can also be called using the call method.

Lamda and Proc differences:

  • Lambda checks the argument passed to it while proc doesn't. Proc will just ignore the additional argument if too many arguments are passed and proc will assume the remaining argument as nil if too few arguments are passed.

    • greet = Proc.new { |greeting, name| puts "Say #{greeting} to #{name}" }
    greet.call("Hello", "Mia", "Morning", "Johnny") #=> "Say Hello to Mia"

    greet = lambda { |greeting, name| puts "Say #{greeting} to #{name}" }
    greet.call("Hello", "Mia", "Morning", "Johnny") #=> wrong number of arguments (4 for 2)

  • When the proc or lambda contains a return statement, lambda will return control back to the calling function and executes the next statement after the lambda call. Proc on the other hand, will exit out of the calling function without executing the next statement after the proc call.

    • def proc_test
    greet = Proc.new { return "Hello from Proc" }
    greet.call
    "In proc_test method"
    end
    p proc_test #=> "Hello from Proc"

    def lambda_test
    greet = lambda { return "Hello from lambda" }
    greet.call
    "In lambda_test method"
    end
    p lambda_test #=> "In lambda test method"
