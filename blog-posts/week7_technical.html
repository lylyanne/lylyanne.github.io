SQL vs NoSQL

20 September 2014

SQL NoSQL
Developed in 1970s Developed in 2000s
Relational database in the form of tables, rows, and columns Document based database
Not designed for agile development where changes happen often and iteratively Designed for agile development
Vertical scaling (buy more RAM or move database to a more powerful server) since the entire database has to be saved on a single server Horizontal scaling (buy more server to get more power)
Requires a fixed schema Doesn't require a predefined schema
